## 技术决策记录

### 1. 为什么选择智谱AI而不是OpenAI?

**决策**: 使用智谱AI GLM-4-Flash

**理由**:
1. ✅ 国内访问稳定,无需代理
2. ✅ 价格更优惠,有免费额度
3. ✅ 中文理解能力强
4. ✅ 响应速度快(Flash版本)
5. ✅ API兼容OpenAI格式,易于切换

**权衡**:
- OpenAI模型能力更强,但国内访问不稳定
- 智谱AI已足够满足客服场景需求

### 2. 为什么使用dotenv而不是配置文件?

**决策**: 使用dotenv管理环境变量

**理由**:
1. ✅ 安全性高,避免敏感信息泄露
2. ✅ 灵活性强,支持不同环境
3. ✅ 行业标准,符合12-factor app
4. ✅ 易于部署,支持容器化

**注意事项**:
- .env文件必须加入.gitignore
- 生产环境应使用密钥管理服务

### 3. 为什么选择axios而不是fetch?

**决策**: 使用axios作为HTTP客户端

**理由**:
1. ✅ 支持超时配置
2. ✅ 自动JSON转换
3. ✅ 请求/响应拦截器
4. ✅ 更好的错误处理
5. ✅ 广泛使用,社区成熟

### 4. 为什么使用TypeScript而不是JavaScript?

**决策**: 使用TypeScript

**理由**:
1. ✅ 类型安全,减少运行时错误
2. ✅ 更好的IDE支持
3. ✅ 代码可维护性高
4. ✅ 大型项目必备
5. ✅ 易于重构

## 问题与解决方案

### 问题1: 端口被占用

**现象**:
```
Error: listen EADDRINUSE: address already in use :::3000
```

**原因**: 多次启动服务,端口未释放

**解决方案**:
```bash
# Windows
taskkill /F /IM node.exe

# Linux/Mac
killall node
```

**预防措施**:
- 使用进程管理工具(PM2)
- 添加端口占用检测

### 问题2: 中文编码显示

**现象**: JSON响应中的中文显示为Unicode转义

**原因**: JSON.stringify默认转义非ASCII字符

**影响**: 仅影响显示,不影响功能

**解决方案**:
- 使用正确的UTF-8解码
- 测试脚本已处理编码问题

### 问题3: 环境变量加载顺序

**现象**: 配置文件读取不到环境变量

**原因**: 模块导入顺序问题

**解决方案**:
```typescript
// server.ts - 必须最先导入
import 'dotenv/config';
import App from './index';
```

**教训**: 环境变量必须在所有其他模块之前加载
